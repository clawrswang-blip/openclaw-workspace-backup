## SFC Uber 深度品类分析项目（下午场）

### 需求
Rishon 上传了两个 Uber transaction Excel 文件，要求在现有 HTML 基础上增加：
1. 按店铺进行深度品类细分（一级+二级分类）
2. 颗粒度集中在产品维度
3. 排除「元初無紡布購物袋」品类
4. 完善 Dashboard 显示

### 技术难点
- HTML 文件中数据量大（数千行），直接嵌入 JSON 存在特殊字符转义问题
- 多次生成失败：Base64+Zlib 方式、字符串拼接方式均因转义失败
- 最终方案：直接用 Python json.dumps 生成 JSON，通过字符串替换嵌入 HTML

### 最终方案
生成文件：`/Users/clawrs/Desktop/sfc-uber-product-analysis.html`
大小：~394KB
数据验证：
- D.Sl: 182行（门店×LV1）
- D.Sll: 519行（门店×LV1×LV2）
- D.Prod: 1680行（产品明细）
- D.Lv1g: 19个一级品类
- D.Lv2g: 89个二级品类
- D.Brand: 29个品牌（新增维度）
- D.Sum: 10个门店

### 6个分析维度
1. **LV1全局概览** — 19个品类卡片+完整表格
2. **门店×LV1** — 门店+LV1二级联动筛选
3. **门店×LV1×LV2** — 三级联动（选LV1自动重置LV2选项）
4. **LV2全局概览** — 89个二级品类跨店横向对比
5. **品牌分析** — 29个品牌表现（新增维度）
6. **产品明细** — 1680个SKU，支持门店+LV1+LV2+关键词搜索

### 数据概览
- 总销售额：$15,829.49 CAD
- 总毛利：$5,145.29 CAD
- 整体毛利率：32.5%
- 已排除：無紡布購物袋（220条记录）
- 分析范围：2,502笔交易，806个产品，10家门店

### 教训
- HTML 内嵌大型 JSON 数据时，`replace()` 方法极易失败（JSON 内含特殊字符破坏 JS 语法）
- 正确做法：Python 生成完整 HTML 字符串，所有数据直接通过字符串替换嵌入
- Base64 编码不是必须的，干净的 json.dumps 输出可以直接嵌入 JS
