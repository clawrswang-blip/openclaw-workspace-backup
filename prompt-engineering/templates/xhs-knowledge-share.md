# xhs-knowledge-share — 干货知识分享

## 触发条件
- 你想建立"专家感"或"可信赖感"
- 目标：让人收藏、转发、评论"学到了"
- 输出形式：知识增量 + 实用价值

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| topic | 知识主题 | 如何挑选有机蔬菜 |
| expertise_level | 受众水平 | 小白 / 有基础 / 专业 |
| key_takeaway | 核心收获（1句话） | 学会看这3个标签，不再被坑 |
| format | 格式偏好 | 清单式 / 故事式 / 对比式 |
| business_relevance | 和业务的关联 | 超市选品 / 会员服务 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你写的小红书知识分享，不是"搬运工"，是"翻译官"——把专业的东西翻译成让人"哇，原来如此"的洞察。

**第一步：毒辣之眼 — 这个话题最大的误解是什么？**

在动笔之前，先问：
- 大多数人对这个话题的误解是什么？（他们以为的 X，其实不是 X）
- 真正重要的东西，90%的人不知道的是什么？
- 如果只记住一个东西，是什么？

**第二步：知识落差设计**

知识分享的本质是"落差"：
- 先说"大多数人的做法"（错的）
- 然后说"正确做法是什么"
- 最后说"为什么这样是对的"

不要给百科全书式的罗列——给"这一个洞见，值千金"的感觉。

**第三步：实用锚点**

每个知识点必须有：
- 可操作的具体步骤（不是"要多练习"，是"第1步、第2步、第3步"）
- 可验证的判断标准（不是"要选好的"，是"满足X就是好的"）
- 能直接套用的场景（不是"这个知识很有用"，是"下次你遇到X就用这个"）

**第四步：输出结构**

```
[开头钩子] — 直接说出误解
  "90%的人在XXX，但其实..." 
  或者："99%的人不知道，XXX其实是这样运作的"

[核心洞见] — 一句话概括
  {key_takeaway}

[知识拆解] — 3-5个要点
  每个要点：
  - 常见误区（先破）
  - 正确理解（再立）
  - 底层逻辑（为什么）

[实用清单/步骤] — 可截图保存
  如果是清单式，给出可直接执行的操作

[结尾升华] — 从知识到认知
  这件事背后反映的是什么？
  对{topic}的理解应该升级到什么维度？
```

**第五步：自检清单**

- [ ] 受众（{expertise_level}）能看懂吗？
- [ ] 最大的误解被点破了吗？
- [ ] 有没有可操作的具体步骤？
- [ ] 读完有没有"学到了"的感觉？
- [ ] 和{business_relevance}的关联自然吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【XHS类·真实感强化】
□ 是否像真实用户的分享，而不是软文？
□ 是否有「踩坑」「后悔」「惊喜」这类真实情绪？
□ 是否有只有真实用过才知道的细节？
□ 图片描述是否符合小红书风格的视觉语言？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
