# cross-cultural-insight — 跨文化洞察

## 触发条件
- 你需要理解不同文化背景下的行为/思维
- 目标：不是"知道差异"，是"理解根源"
- 场景：出海、市场扩张、跨文化合作

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| culture_a | 文化A | 中国大陆商业文化 |
| culture_b | 文化B | 北美商业文化 |
| context | 场景 | 商务谈判 / 团队合作 |
| specific_situation | 具体情境 | 第一次见面怎么建立信任 |
| goal | 目标 | 找到有效沟通的方式 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你做的跨文化洞察，不是"中国人这样，外国人那样"，是"这些差异背后的系统是什么"。

---

**文化A：** {culture_a}
**文化B：** {culture_b}
**场景：** {context}
**具体情境：** {specific_situation}
**目标：** {goal}

---

**第一步：毒辣之眼 — 跨文化最大的陷阱是什么？**

在动笔之前，先问：
- 最大的陷阱不是"不了解对方"，是"用己度人"——你怎么检视自己？
- "文化差异"是真实存在，还是你给自己找的借口？
- {specific_situation}里，双方各自真正在意的是什么？

**第二步：文化维度分析**

用系统性框架分析：

| 维度 | {culture_a} | {culture_b} |
|------|-----------|-----------|
| 权力距离 | 高/低 | 高/低 |
| 个人vs集体 | ? | ? |
| 不确定性规避 | ? | ? |
| 长期vs短期 | ? | ? |
| 表达方式 | 直接/含蓄 | 直接/含蓄 |

**第三步：{specific_situation}分析**

在这个{specific_situation}里：

```
[{culture_a}的视角]
  - 他们怎么看待这个场景？
  - 他们的期待是什么？
  - 什么是"正常"？

[{culture_b}的视角]
  - 他们怎么看待这个场景？
  - 他们的期待是什么？
  - 什么是"正常"？

[真实冲突点]
  - 双方期待的不同在哪里？
  - 这背后的文化根源是什么？
```

**第四步：有效策略**

不是"谁对谁错"：

**策略1：框架转换**
- 从{culture_a}框架转换到{culture_b}框架
- 关键动作

**策略2：寻找共同点**
- {context}里，什么是双方都关心的？
- 用共同点建立基础

**策略3：具体技巧**
- 在{specific_situation}里，具体可以怎么做

**第五步：闭环验证**

- 价值创造：这个洞察能帮你做什么？
- 价值传递：你怎么用这个洞察？
- 价值捕获：跨文化理解带来什么竞争优势？

**第六步：输出结构**

```
[跨文化主题]
  {culture_a} vs {culture_b}
  场景：{context}

[核心洞察] — 一句话
  [这个跨文化比较最重要的点]

[文化维度对比]
  [表格]

[{specific_situation}分析]
  [双方视角]
  [冲突点]

[有效策略]
  策略1
  策略2
  策略3

[实践建议]
  [具体的可操作建议]
```

**第七步：自检清单**

- [ ] 不是"刻板印象"而是真的洞察吗？
- [ ] 双方视角都被公平对待了吗？
- [ ] 有具体可操作的建议吗？
- [ ] {goal}能达到吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【个人成长类·真实挣扎强化】
□ 是否承认了困难和不确定性？
□ 是否有具体的试错过程？
□ 是否有「我以为...但其实...」的反转？
□ 是否给了「还没解决」的部分？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
