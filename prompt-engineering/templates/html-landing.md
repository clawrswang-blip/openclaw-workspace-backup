# html-landing — HTML/Landing Page 创作

## 触发条件
- 你需要创建一个可运行的 Landing Page
- 目标：直接可用的 HTML/CSS/JS
- 关联：html-ppt skill, frontend-design-3

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| page_type | 页面类型 | 营销落地页 / 产品展示 / 活动页 |
| brand_name | 品牌名 | Sungiven Foods |
| brand_color | 品牌色 | #2D5A27 |
| main_content | 核心内容 | Hero区文案、CTA |
| tech_stack | 技术栈 | 纯HTML / React / Vue |
| responsive | 响应式 | 是 / 否 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你写的 Landing Page，不是"好看的图"，是"能转化的人机界面"。

**第一步：毒辣之眼 — 这个页面最可能失败的地方是什么？**

在动笔之前，先问：
- 用户（{page_type}）第一秒会注意到什么？（注意力有限）
- 最大的"转化漏斗流失"在哪里？（不是"设计好不好看"，是"能不能完成目标"）
- 移动端 vs 桌面端的行为差异考虑了吗？

**第二步：结构设计**

Landing Page 的核心结构：
```
[Hero] — 钩子+CTA
[Social Proof] — 信任建立
[Features/Benefits] — 价值展示
[How It Works]（如需）— 降低认知门槛
[Testimonials]（如需）— 社会认同
[CTA] — 最后的转化机会
[Footer] — 必要信息
```

**第三步：技术要求**

{tech_stack} + {responsive}：
- 性能：首屏加载 < 2秒
- SEO：语义化标签
- 可访问性：ARIA标签

**第四步：输出要求**

生成可直接运行的代码：
- HTML 结构清晰
- CSS 内联或单独文件
- JS 最小化，按需加载
- 图片使用占位符或 CDN 链接

**第五步：自检清单**

- [ ] 结构符合 {page_type} 的标准吗？
- [ ] CTA 够明显吗？（不只一个）
- [ ] 移动端（{responsive}）体验考虑了吗？
- [ ] 性能考虑了吗？
- [ ] 代码可直接运行吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【UI/UX设计类·工程思维强化】
□ 是否考虑了可扩展性和维护性？
□ 是否说明了技术选型的权衡？
□ 是否有错误处理和边界条件？
□ 注释是否说明了「为什么」而不是「是什么」？

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
