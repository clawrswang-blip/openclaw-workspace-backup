# brainstorming — 头脑风暴

## 触发条件
- 你需要一个创意的爆发
- 目标：量先于质，不要过滤
- 关联：marketing-ideas skill

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| problem_statement | 问题陈述 | 如何提升会员复购率 |
| brainstorming_type | 类型 | 发散型 / 收敛型 / 逆向型 |
| participants | 参与者 | 1人Solo / 团队 |
| quantity_goal | 目标数量 | 20个想法 |
| constraints | 约束条件 | 预算C$5000 / 3个月内 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你做头脑风暴，不是"想几个点子"，是"先破坏性发散，再理性收敛"。

**类型：** {brainstorming_type}

**目标数量：** {quantity_goal}
**约束条件：** {constraints}

**第一步：发散阶段（不评判）**

规则：{quantity_goal}个想法，不筛选，不批评

方法：
- 逆向思考："怎么让复购率降到0？"
- 极端想法："如果预算无限呢？"
- 跨界借鉴："其他行业怎么做会员复购？"
- 否定再翻转："不要会员卡"→"随时随地会员"

输出：{quantity_goal}个原始想法（raw list）

**第二步：分类阶段**

把{quantity_goal}个想法分组：
- 同类归并
- 明显不可行的标记
- 看起来有潜力的标记

**第三步：收敛阶段**

{quantity_goal}→5个精选：

评估维度：
- 可执行性：{constraints}下能做吗？
- 影响力：这个想法能带来多大改变？
- 创新性：是"老办法"还是"新思路"？
- 风险：最大的风险是什么？

**第四步：深化5强**

每个精选想法：
- 核心逻辑
- 执行路径
- 需要的资源
- 潜在风险

**第五步：输出结构**

```
[问题陈述]
  {problem_statement}
  类型：{brainstorming_type}

[发散阶段]
  [原始想法列表，共{quantity_goal}个]

[分类与筛选]
  [分组]
  [不可行标记]
  [有潜力标记]

[收敛5强]
  1. [想法名]
     - 核心逻辑
     - 执行路径
     - 风险

  2. ...

[推荐执行顺序]
  [优先级]
```

**第六步：自检清单**

- [ ] 发散阶段真的没评判吗？
- [ ] 逆向思考用了吗？
- [ ] {quantity_goal}个想法够吗？
- [ ] 收敛有标准吗？（不是随便选）
- [ ] 约束条件（{constraints}）考虑了？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
