# code-generation — 代码/脚本生成

## 触发条件
- 你需要一段代码（Python/JS/Shell等）
- 目标：能直接运行，不是"示例代码"
- 关联：coding-agent skill

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| language | 语言 | Python |
| task | 任务 | 数据处理脚本 |
| input_data | 输入数据 | CSV文件 |
| output_format | 输出格式 | JSON |
| performance_req | 性能要求 | 处理10万行<10秒 |
| error_handling | 错误处理 | 需要 / 不需要 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你写的代码，不是"能跑就行"，是"能维护、可扩展、有边界"。

**第一步：毒辣之眼 — 代码最大的问题是什么？**

在动笔之前，先问：
- 最大的"不知道边界"在哪里？（输入什么会挂？）
- 性能（{performance_req}）的瓶颈在哪里？
- {error_handling}需要处理什么错误？（空值、异常值、格式错误）

**第二步：代码结构**

```
[输入验证]
  - 数据格式
  - 边界条件
  - 异常输入处理

[核心逻辑]
  - 主要功能实现
  - 算法选择

[输出处理]
  - {output_format}
  - 格式化

[错误处理]（如需要）
  - {error_handling}
  - 日志
```

**第三步：代码规范**

- 变量命名：见名知意
- 函数：单一职责
- 注释：解释"为什么"，不解释"是什么"
- 测试：关键逻辑有验证

**第四步：自检清单**

- [ ] 能直接运行吗？（不是伪代码）
- [ ] 边界条件处理了吗？
- [ ] {performance_req}能满足吗？
- [ ] {error_handling}够吗？
- [ ] 代码可读吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
