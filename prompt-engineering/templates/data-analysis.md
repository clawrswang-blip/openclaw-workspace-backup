# data-analysis — 数据分析报告

## 触发条件
- 你有数据需要分析
- 目标：从数据中找洞察，不是做图表搬运工
- 关联：Luna Data Analysis skill

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| data_type | 数据类型 | 销售数据 / 用户行为数据 |
| data_source | 数据来源 | Google Analytics / 内部CRM |
| analysis_goal | 分析目标 | 找出下降原因 |
| key_metrics | 关键指标 | 月活、转化率、客单价 |
| time_range | 时间范围 | 过去6个月 |
| comparison_baseline | 对比基准 | 去年同期 / 上个月 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你做的数据分析，不是"数字摆出来"，是"告诉我发生了什么、为什么、怎么办"。

**第一步：毒辣之眼 — 你看到的数据，最可能误导你的是什么？**

在动笔之前，先问：
- 相关性 vs 因果性：数字显示A和B相关，但真的有关系吗？
- 幸存者偏差：你看到的是全部数据，还是只是幸存者？
- 数据噪音：哪个波动是真实的，哪个只是噪音？

**第二步：分析框架**

```
[描述性分析] — 发生了什么
  {key_metrics}在{time_range}的表现
  vs {comparison_baseline}

[诊断性分析] — 为什么发生
  找原因：假设检验
  排除巧合：统计显著性

[探索性分析] — 还有什么
  异常发现
  相关性
  新假设
```

**第三步：闭环验证**

- 数据可信吗？（来源、采集方法）
- 分析假设成立吗？（A导致B，不是C导致B）
- 结论能指导行动吗？（不是"下降了10%"，是"因为X，需要做Y"）

**第四步：输出结构**

```
[执行摘要] — 一页纸
  核心发现（3-5条）
  建议（2-3条）

[数据概览]
  {time_range}关键指标
  vs {comparison_baseline}

[核心发现]
  每个发现：
  - 数字
  - 解读
  - 置信度
  - 反事实检验

[异常分析]（如有）
  异常点
  可能原因

[建议]
  基于数据的具体行动
  优先级
```

**第五步：自检清单**

- [ ] 分析回答了{analysis_goal}吗？
- [ ] 数字背后的"为什么"找到了吗？
- [ ] 相关性 vs 因果性区分清楚了吗？
- [ ] 建议具体吗？（不是"加强运营"）
- [ ] 数据可信度评估了吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【深度分析类·穿透力强化】
□ 是否回答了「为什么会这样？」而不是「是这样的」？
□ 是否有第二层、第三层因果分析？
□ 是否提出了反直觉的洞察？
□ 是否明确了「谁会不同意这个结论」？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
