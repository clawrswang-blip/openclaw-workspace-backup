# customer-research — 用户研究

## 触发条件
- 你需要了解你的用户/客户
- 目标：不只是"他们说什么"，是"他们为什么这样做"
- 场景：产品迭代、定位调整、新市场进入

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| user_segment | 用户群体 | Sungiven会员 |
| research_questions | 研究问题（2-3个） | 为什么不续费？ |
| research_method | 研究方法 | 访谈 / 问卷 / 数据分析 |
| output_type | 输出类型 | 用户画像 / 需求报告 / 洞察 |
| business_goal | 业务目标 | 提升续费率 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你做的用户研究，不是"收集反馈"，是"读懂他们没说出口的需求"。

**第一步：毒辣之眼 — 用户研究最大的陷阱是什么？**

在动笔之前，先问：
- 用户说的 vs 用户做的，差异在哪里？（言行不一）
- 用户最大的"说不清楚"是什么？（他们的需求在冰山以下）
- 这个研究（{research_goal}）要改变什么？（不是收集数据，是改变决策）

**第二步：研究设计**

{research_method}要回答的问题：
- 定量：有多大比例的用户有这个问题？
- 定性：用户怎么描述这个问题？
- 探索：这个研究要发现什么新东西？

**第三步：洞察提炼**

从数据到洞察：
```
表面数据 → 行为模式 → 深层动机 → 洞察
"不续费"  → "用完优惠券后流失" → "没有建立使用习惯" → "需要在30天内建立粘性"
```

**第四步：输出结构**

```
[研究背景]
  {user_segment}
  {research_questions}
  {research_method}

[核心洞察] — 一句话
  [这个研究最重要的发现]

[用户画像]（如需）
  人口统计
  行为特征
  需求/动机
  痛点

[关键发现]
  每个发现：
  - 现象（用户做什么）
  - 本质（为什么这样做）
  - 证据（数据/访谈引用）

[建议]
  基于洞察的具体建议
  不是"优化体验"，是"在30天内建立使用习惯"
```

**第五步：自检清单**

- [ ] 有没有"言行不一"的洞察？
- [ ] 有没有"用户说不清楚"的深层需求？
- [ ] 洞察能直接指导{research_goal}吗？
- [ ] 建议具体吗？（不是"加强用户运营"）

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【深度分析类·穿透力强化】
□ 是否回答了「为什么会这样？」而不是「是这样的」？
□ 是否有第二层、第三层因果分析？
□ 是否提出了反直觉的洞察？
□ 是否明确了「谁会不同意这个结论」？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
