# m&a-analysis — 商业并购/合作分析

## 触发条件
- 你在考虑一个并购或深度合作机会
- 目标：不只是"合不合法"，是"值不值"
- 场景：收购、合并、战略投资、深度合作

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| deal_type | 交易类型 | 收购 / 合并 / 战略投资 / 深度合作 |
| target_name | 目标公司/项目 | ABC供应链公司 |
| deal_rationale | 交易动机 | 补强供应链 |
| valuation_range | 估值范围 | C$500-800万 |
| integration_challenge | 整合挑战 | 团队融合、系统对接 |
| timeline | 时间线 | 3个月闭环 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你做的并购/合作分析，不是"数字游戏"，是"两个组织的化学反应"。

---

**第一步：毒辣之眼 — 并购最大的失败是什么？**

在动笔之前，先问：
- 大多数并购失败不是因为"买贵了"，是因为"合不来"——为什么？
- {deal_rationale}是真的吗？（还是你在合理化？）
- 你最担心的{integration_challenge}是什么？（你最不想面对的，往往是关键）

**第二步：战略契合分析**

```
[战略层]
  - {deal_rationale}成立吗？
  - 目标{target_name}的战略是什么？
  - 两者能对齐吗？

[运营层]
  - {integration_challenge}的根源是什么？
  - 整合成本有多高？
  - 需要多长时间？

[文化层]
  - 双方的文化兼容吗？
  - 关键人员会留下吗？
  - 决策机制怎么合并？
```

**第三步：估值与价格**

{valuation_range}合理吗？

分析维度：
- 资产估值：净资产、重置成本
- 收益估值：PE、EV/EBITDA
- 市场估值：可比交易、可比公司
- 协同估值：整合后的价值增量

**第四步：风险矩阵**

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 整合失败 | 中 | 高 | ... |
| 人员流失 | 高 | 高 | ... |
| 估值过高 | 中 | 高 | ... |

**第五步：决策检查清单**

- [ ] {deal_rationale}被验证了吗？（不只是"觉得有道理"）
- [ ] 整合挑战（{integration_challenge}）有应对方案吗？
- [ ] 估值（{valuation_range}）有安全边际吗？
- [ ] 关键人员留下有协议吗？
- [ ] 退出机制设计了吗？

**第六步：输出结构**

```
[交易概览]
  {deal_type} / {target_name}
  {valuation_range}

[核心洞察] — 一句话
  [这个交易最大的机会和最大的风险]

[战略契合]
  动机分析
  战略对齐

[尽职调查重点]
  [必须查的3件事]

[估值分析]
  [估值方法]
  [合理区间]

[风险矩阵]
  [表格]

[整合方案]
  {integration_challenge}
  [整合计划]

[建议]
  推进 / 暂停 / 重新谈 / 放弃
```

**第七步：自检清单**

- [ ] {deal_rationale}被独立验证了吗？
- [ ] 整合（{integration_challenge}）有详细计划吗？
- [ ] 估值（{valuation_range}）有安全边际吗？
- [ ] 关键人才能保留吗？
- [ ] 风险可承受吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【深度分析类·穿透力强化】
□ 是否回答了「为什么会这样？」而不是「是这样的」？
□ 是否有第二层、第三层因果分析？
□ 是否提出了反直觉的洞察？
□ 是否明确了「谁会不同意这个结论」？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
