# agent-bot-design — Agent/Bot 开发设计

## 触发条件
- 你需要设计一个 Agent 或 Bot
- 目标：真正能用的，不是"演示版"
- 关联：agent-browser-clawdbot skill

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| agent_name | Agent名称 | Luna运营助手 |
| primary_function | 核心功能 | 监控+预警+报告 |
| user_interaction | 交互方式 | 文字 / 语音 / 混合 |
| platform | 部署平台 | Slack / Discord / 飞书 |
| autonomy_level | 自主程度 | 执行命令 / 建议 / 半自动 |
| data_sources | 数据源 | 数据库 / API / 爬虫 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你设计的Agent，不是"能做很多事"，是"能把一件事做好"。

**第一步：毒辣之眼 — Agent最大的失败是什么？**

在动笔之前，先问：
- Agent最常"做过头"的是什么？（自主行动但出错）
- 用户最怕Agent做什么？（不受控的行为）
- {autonomy_level}的边界在哪里？

**第二步：功能设计**

{primary_function}分解：

```
[核心能力]
  1. [能力1]
     - 输入：什么
     - 输出：什么
     - 触发条件：什么时候

  2. [能力2]
     ...

[边界定义]
  - Agent能做的
  - Agent不能做的
  - 人工确认的节点
```

**第三步：交互设计**

{user_interaction} + {platform}：

```
[命令格式]
  - 触发词
  - 参数格式
  - 示例

[反馈机制]
  - 执行成功：反馈什么
  - 执行失败：反馈什么
  - 需要确认：怎么问

[日志记录]
  - 记录什么
  - 记录在哪里
```

**第四步：技术架构**

{data_sources} + {autonomy_level}：

```
[数据流]
  数据源 → 处理 → 输出 → 用户

[API设计]（如需）
  - 接口定义
  - 认证

[错误处理]
  - 失败重试
  - 降级策略
```

**第五步：输出结构**

```
[Agent概览]
  {agent_name}
  核心功能：{primary_function}

[功能设计]
  [详细]

[交互设计]
  [命令格式]
  [反馈机制]

[技术架构]
  [数据流]
  [API]

[边界与安全]
  - 能做的
  - 不能做的
  - 人工确认节点
```

**第六步：自检清单**

- [ ] 核心功能（{primary_function}）单一吗？
- [ ] 自主程度（{autonomy_level}）有边界吗？
- [ ] 错误处理有吗？
- [ ] 用户知道Agent在做什么吗？
- [ ] 数据源（{data_sources}）都接上了吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【代码/技术类·工程思维强化】
□ 是否考虑了可扩展性和维护性？
□ 是否说明了技术选型的权衡？
□ 是否有错误处理和边界条件？
□ 注释是否说明了「为什么」而不是「是什么」？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
