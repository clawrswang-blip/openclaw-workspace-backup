# business-ideation — 商业想法落地评估

## 触发条件
- 你有一个商业想法需要评估
- 目标：不只是"想"，是"想清楚"
- 场景：创业评估、新项目立项、内部创新

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| business_idea | 商业想法 | 温哥华有机食品订阅盒 |
| your_background | 你的背景/资源 | 供应链、食品行业 |
| time_horizon | 时间维度 | 3个月 / 1年 / 3年 |
| risk_tolerance | 风险承受 | 低 / 中 / 高 |
| investment_capacity | 可投入资源 | 时间/金钱/团队 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你评估商业想法，不是"这个想法好不好"，是"这个想法在你手里能不能成"。

---

**商业想法：** {business_idea}
**你的背景/资源：** {your_background}
**时间维度：** {time_horizon}
**风险承受：** {risk_tolerance}
**可投入资源：** {investment_capacity}

---

**第一步：毒辣之眼 — 你没想清楚的是什么？**

在动笔之前，先问：
- 这个想法最大的"不成立"是什么？（什么时候会失败？）
- 你（{your_background}）凭什么能做成功？（护城河是什么？）
- 时间（{time_horizon}）够不够？（3个月和3年的玩法完全不同）

**第二步：想法解剖**

```
[问题层]
  - 解决什么人的什么问题？
  - 现有解决方案是什么？
  - 你的差异化在哪里？

[价值层]
  - 用户愿意付多少钱？为什么？
  - 市场有多大？（TAM/SAM/SOM）
  - 规模化路径是什么？

[执行层]
  - 最小可行产品是什么？
  - 第一批用户从哪里来？
  - 关键假设是什么？

[风险层]
  - 最大的3个风险
  - 每个风险的应对方案
  - {risk_tolerance} 下能承受吗？
```

**第三步：闭环验证**

- 价值创造：真的解决问题了吗？（不是伪需求）
- 价值传递：用户怎么知道？怎么相信？
- 价值捕获：钱怎么流向你？利润结构？

**第四步：十年视角**

如果{time_horizon}是3年：
- 3年后这个市场会是什么样子？
- 如果不做这个，3年后会后悔吗？
- 最坏情况是什么？发生概率？

**第五步：输出结构**

```
[想法概览]
  {business_idea}

[核心洞察] — 一句话
  [这个想法最大的机会和最大的风险]

[问题与价值]
  问题定义
  差异化
  市场分析

[执行路径]
  MVP
  增长路径
  关键假设

[风险评估]
  Top 3风险
  应对方案

[时间轴]
  {time_horizon}
  里程碑

[建议]
  做 / 不做 / 改造后再评估
```

**第六步：自检清单**

- [ ] 问题是真的吗？（不是伪需求）
- [ ] 差异化存在吗？（不是"我也能做"）
- [ ] 闭环成立吗？（创造→传递→捕获）
- [ ] {risk_tolerance} 能承受吗？
- [ ] 时间（{time_horizon}）够吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【商业策略类·闭环强化】
□ 是否明确了「谁来买单、凭什么、持续多久」？
□ 风险点是否说清楚了？
□ 是否有清晰的行动边界？
□ 是否给出了「如果这样就失败」的预警？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
