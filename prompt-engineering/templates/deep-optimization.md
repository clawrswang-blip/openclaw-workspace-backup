# deep-optimization — 深度优化评审

## 触发条件
- 你有一个方案/产品/流程需要优化
- 目标：找到最大杠杆点，不是"小修小补"
- 关联：seo-audit, page-cro

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| subject | 评审对象 | 会员注册流程 |
| optimization_type | 类型 | 转化优化 / 体验优化 / 效率优化 |
| current_metrics | 当前指标 | 注册转化率12% |
| target_metrics | 目标指标 | 20% |
| constraint | 约束 | 不能改变核心产品逻辑 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你做的深度优化，不是"哪里不对改哪里"，是"找到最大杠杆点，一击必中"。

**第一步：毒辣之眼 — 优化最大的陷阱是什么？**

在动笔之前，先问：
- 最大的优化机会不在"细节"，在"结构"——在哪里？
- 你以为是"注册流程"的问题，但真正的问题在哪里？
- {constraint}背后是"真约束"还是"你设想的约束"？

**第二步：现状诊断**

```
[问题定位]
  - 哪里出了{current_metrics}→{target_metrics}的差距？
  - 用户/客户在哪里流失？
  - 行为数据显示什么？

[归因分析]
  - 假设：转化低的3个原因
  - 每个原因的置信度
  - 每个原因的验证方法

[约束评估]
  - {constraint}是真实的吗？
  - 打破这个约束的成本是什么？
  - 值得打破吗？
```

**第三步：杠杆点识别**

不是"每个问题都同等重要"：

| 问题 | 影响度 | 解决成本 | 杠杆率 |
|------|--------|---------|--------|
| 问题A | 高 | 低 | ★★★★★ |
| 问题B | 高 | 高 | ★★★ |
| 问题C | 低 | 低 | ★★ |

杠杆率 = 影响度 / 解决成本

**第四步：优化方案**

针对Top 3杠杆点：

每个方案：
- 核心改动
- 预期效果
- 实施成本
- 风险

**第五步：验证设计**

每个优化方案要设计"验证方式"：
- A/B测试怎么设计？
- 成功/失败的判定标准？
- 需要多少样本？

**第六步：输出结构**

```
[优化概览]
  {subject}
  {current_metrics} → {target_metrics}

[核心洞察] — 一句话
  [真正的问题在哪里]

[现状诊断]
  问题定位
  归因分析

[杠杆点排名]
  [表格]

[优化方案]
  方案1
  方案2
  方案3

[验证设计]
  [每个方案的测试方案]
```

**第六步：自检清单**

- [ ] 找到真正的杠杆点了吗？
- [ ] 归因有证据吗？（不是猜测）
- [ ] 约束（{constraint}）评估了吗？
- [ ] 有验证设计吗？（不是拍脑袋）
- [ ] 方案可执行吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【商业策略类·闭环强化】
□ 是否明确了「谁来买单、凭什么、持续多久」？
□ 风险点是否说清楚了？
□ 是否有清晰的行动边界？
□ 是否给出了「如果这样就失败」的预警？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
