# cold-email-sequence — Cold Email 序列

## 触发条件
- 你需要一序列Cold Email（3-5封）
- 目标：让陌生人回复（不是直接成交）
- 场景：B2B销售、合作关系、职位申请

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| recipient_role | 收件人角色 | 中小企业老板 |
| recipient_pain | 收件人的痛点 | 知道数字化重要但不知道怎么开始 |
| your_value | 你的价值（1句话） | 帮企业用AI提升效率 |
| email_sequence_type | 序列类型 | 价值主张型 / 问题发现型 |
| cta_per_email | 每封邮件的CTA | 第一封：回复看法 / 第二封：预约通话 |
| sequence_length | 序列长度 | 3封 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你写的Cold Email，不是"我有什么"，是"我发现你有个问题，我能解决"。

---

**第一步：毒辣之眼 — 你没说的是什么？**

在动笔之前，先问：
- 收件人（{recipient_role}）每天收到多少封这样的邮件？（比你想象的多）
- 他们凭什么打开这封？凭什么回复？（注意力稀缺）
- 你最大的"差异化"是什么？（不是"我们很专业"，是"我看到你没看到的东西"）

**第二步：序列设计**

{sequence_length}封邮件不是"重复发同样的内容"：

| 封次 | 目的 | 内容 |
|------|------|------|
| 第1封 | 打开+引发好奇 | 痛点共鸣，不给答案 |
| 第2封 | 深化问题 | 给洞察，给案例 |
| 第3封 | CTA | 明确的行动召唤 |

**第三步：每封邮件的公式**

```
[Subject] — 30字符内，引发好奇
  不是"合作机会"，是"[痛点相关]"

[Preview Text] — 90字符内，补充Subject

[Body]
  - 开头：直接进入痛点
  - 中段：洞察/问题/案例
  - 结尾：{cta_per_email}

[签名]
  简短、有身份、不夸张
```

**第四步：自检清单**

- [ ] Subject能让人打开吗？
- [ ] 每封邮件的"不说"是什么？（不能一次说完）
- [ ] CTA（{cta_per_email}）够低门槛吗？
- [ ] 序列有节奏感吗？（不是重复，是递进）
- [ ] 够简短吗？（越短越好）

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【商业策略类·闭环强化】
□ 是否明确了「谁来买单、凭什么、持续多久」？
□ 风险点是否说清楚了？
□ 是否有清晰的行动边界？
□ 是否给出了「如果这样就失败」的预警？

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
