# dashboard-design — Dashboard 设计需求

## 触发条件
- 你需要设计一个数据看板
- 目标：让数据驱动决策，不是"看起来很酷"
- 场景：业务监控、管理驾驶舱、产品指标

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| dashboard_purpose | Dashboard用途 | 会员运营周报 |
| primary_users | 主要用户 | 运营团队 |
| key_questions | 核心问题（3-5个） | 本周新注册多少？转化率？ |
| data_sources | 数据来源 | CRM、GA、支付系统 |
| refresh_frequency | 更新频率 | 实时 / 每日 / 每周 |
| platform | 平台 | Looker / Metabase / 自建 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你设计的Dashboard，不是"把所有数字放上去"，是"让决策者一眼看清关键"。

---

**主要用户：** {primary_users}
**核心问题：** {key_questions}
**数据来源：** {data_sources}
**更新频率：** {refresh_frequency}
**平台：** {platform}

**第一步：毒辣之眼 — Dashboard最大的失败是什么？**

在动笔之前，先问：
- {primary_users}看到Dashboard的第一反应是什么？（"这些数字是什么意思？"）
- 最大的问题是"数字堆砌"——谁看谁头疼
- Dashboard不是"展示数据"，是"压缩信息，辅助决策"

**第二步：信息架构**

{key_questions}是最重要的——每个问题对应一个视图/模块。

设计原则：
- 一级视图：最高频的3个问题，一眼看清
- 二级视图：次重要的问题，需要点击展开
- 三级视图：长尾问题，放附录/详情页

**第三步：指标设计**

每个指标要回答：
- 这个指标是什么？（定义）
- 正常范围是多少？（基准）
- 异常了怎么办？（Action）

**第四步：输出结构**

```
[Dashboard概览]
  用途：{dashboard_purpose}
  用户：{primary_users}

[核心问题映射]
  问题1 → 视图/指标
  问题2 → 视图/指标
  ...

[信息架构]
  一级视图（最重要的）
  二级视图（次重要的）
  三级视图（详情/附录）

[指标定义]
  指标1：定义 | 基准 | 异常Action
  指标2：...

[数据源说明]
  {data_sources}

[平台建议]（如需）
  {platform}
```

**第五步：自检清单**

- [ ] {primary_users}能一眼看清最重要的吗？
- [ ] 每个{key_questions}都有对应的指标吗？
- [ ] 指标有"异常怎么办"的Action吗？
- [ ] 信息层级清晰吗？
- [ ] {refresh_frequency}能满足决策需求吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【深度分析类·穿透力强化】
□ 是否回答了「为什么会这样？」而不是「是这样的」？
□ 是否有第二层、第三层因果分析？
□ 是否提出了反直觉的洞察？
□ 是否明确了「谁会不同意这个结论」？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
