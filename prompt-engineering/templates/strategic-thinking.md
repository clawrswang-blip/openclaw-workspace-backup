# strategic-thinking — 高维度战略思考

## 触发条件
- 你需要战略性思考（不是战术执行）
- 目标：看10年后，而不是今天
- 场景：重大决策、方向调整、长期规划

## 输入变量

| 变量 | 说明 | 示例 |
|------|------|------|
| decision_topic | 决策主题 | 是否全面转型AI驱动运营 |
| time_horizon | 时间维度 | 3年 / 5年 / 10年 |
| key_constraints | 关键约束 | 资金 / 团队 / 时间 |
| stakeholders | 利益相关方 | 创始人 / 团队 / 投资人 |
| success_criteria | 成功标准 | 如何定义成功 |

## 核心 Prompt

===新增===
【质量锚点：Top 3% 标准】
- 不是「够用」，是「让人想收藏」
- 不是「说明白了」，是「看完觉得值」
- 不是「通用答案」，是「这个细分领域的深层洞察」
- 不是「表面总结」，是「别人没看到的维度」
- 所有数据必须有来源，所有观点必须有依据
===新增===

你做战略思考，不是"怎么解决这个问题"，是"10年后回头看，这个决策是对的吗"。

**决策主题：** {decision_topic}
**时间维度：** {time_horizon}
**关键约束：** {key_constraints}
**利益相关方：** {stakeholders}
**成功标准：** {success_criteria}

**第一步：毒辣之眼 — 你没想10年的地方是什么？**

在动笔之前，先问：
- 10年后（{time_horizon}）这个领域会是什么样子？
- 如果不做这个决定，10年后会后悔吗？
- 这个决定最大的"反事实"是什么？（如果做了会怎样？不做会怎样？）

**第二步：战略框架**

```
[第一性原理]
  - 这个决定的根本问题是什么？
  - 什么是不变的？什么是会变的？

[时间维度]
  - 10年后：这个决定对吗？
  - 5年后：里程碑是什么？
  - 今天：第一步是什么？

[利益相关方分析]
  - {stakeholders}
  - 他们的诉求是什么？
  - 如何平衡？

[约束分析]
  - {key_constraints}
  - 哪些是真正的约束？
  - 哪些是假装的约束？
```

**第三步：反事实测试**

"如果这个决策是错的，最可能的原因是什么？"
- 写下3个"错了"的假设
- 每个假设发生的概率？
- 每个假设下，你怎么办？

**第四步：战略选项**

不是"做还是不做"：
- 选项A：全面执行
- 选项B：试点后扩大
- 选项C：暂时不做，监控等待
- 选项D：其他替代方案

每个选项的：
- 优势
- 风险
- {time_horizon}后的结果

**第五步：输出结构**

```
[战略问题]
  {decision_topic}

[核心洞察] — 一句话
  [10年后回头看，什么是最重要的]

[第一性原理]
  [根本问题]

[时间轴]
  - 10年后愿景
  - 5年里程碑
  - 今天的第一步

[利益相关方]
  [分析]

[反事实测试]
  [错了的假设+应对]

[战略选项]
  选项A/B/C/D
  [对比]

[建议]
  [决策]
```

**第六步：自检清单**

- [ ] 有10年（{time_horizon}）视角吗？
- [ ] 反事实测试做了吗？
- [ ] 有多个选项吗？（不是二元对立）
- [ ] 利益相关方（{stakeholders}）考虑了吗？
- [ ] {success_criteria} 清晰吗？

===新增===
【去AI化检查】
□ 是否有具体的个人观察/数据点，而不是泛泛而谈？
□ 是否有不完美的、真实的细节？（AI总爱给完美答案）
□ 句式是否有变化，还是全是「首先...其次...最后...」？
□ 是否有反直觉的洞见，而不是安全的主流观点？
□ 是否避免了「不仅...而且...」「随着...」这类AI最爱用的句式？
□ 是否有情绪温度，还是全程冷静得像产品说明书？
□ 是否有具体的场景/案例，而不是笼统的「比如说...」？
===新增===

===新增===
【商业策略类·闭环强化】
□ 是否明确了「谁来买单、凭什么、持续多久」？
□ 风险点是否说清楚了？
□ 是否有清晰的行动边界？
□ 是否给出了「如果这样就失败」的预警？
===新增===

## 自我迭代检查（必做）

### 产出后自检（Before 输出）
- [ ] 我提供的洞见，读者在其他地方能轻易找到吗？（如果是，降级重写）
- [ ] 我的核心观点能用一句话概括吗？（如果不能，结构有问题）
- [ ] 最有争议的那个点，我有没有回避？（回避了=不够真诚）

### 产出后迭代（After 初稿）
问题 1：标题是否足够吸引，让 80% 的人想点进来？
问题 2：开头是否在 3 秒内抛出「钩子」？
问题 3：是否有至少一个「反直觉」的洞察？
问题 4：结尾是否给出了行动指引或思考延伸，而不是「以上就是...」？
问题 5：是否符合该平台的语言风格？（XHS vs 公众号 vs 知乎 vs 微博）
